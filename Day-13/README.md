# Day 13 - Create AMI from EC2 Instance

**YouTube Video Link:**  
[Watch the solution here](https://www.youtube.com/watch?v=tSLQx_HeRlg)

**Task Summary:**  
In this task, you learn how to create a custom Amazon Machine Image (AMI) from an existing EC2 instance in AWS. An AMI is a snapshot-based template that captures the full state of an instance — OS, software, and configuration — so you can launch identical instances quickly. This is a fundamental technique for building repeatable and scalable cloud infrastructure.

**What you will learn:**  
- What an AMI is and how it serves as a reusable template for launching EC2 instances.  
- How to create a custom AMI from a running or stopped EC2 instance via the AWS Console.  
- How AWS creates EBS snapshots under the hood when an AMI is created.  
- Best practice: Create AMIs from stopped instances to ensure data consistency; use descriptive naming with version tags (e.g., `webserver-v1.0-2024`) for easy identification.
