**YouTube Video Link:**

**Task Summary:**  
In this task, you learn how to create an IAM Group in AWS, which is a collection of IAM users that share the same set of permissions. Instead of assigning policies to individual users, you attach policies to a group and then add users to that group — making permission management much easier to scale. IAM Groups are an essential tool for organizing access control in any AWS environment.

**What you will learn:**  
- What an IAM Group is and how it simplifies permission management compared to per-user policies.  
- How to create an IAM Group, attach a managed or custom policy to it, and add IAM users to it.  
- Common group patterns: `Admins`, `Developers`, `ReadOnly`, `Billing`.  
- Best practice: Always assign permissions to groups rather than individual users — this makes access auditing and revoking much simpler when team membership changes.
